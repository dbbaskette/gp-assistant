## ‚ú® Why It Shines

- **üß† Greenplum-native RAG** ‚Äî Tailored prompt templates, doc indexing, and version-aware context keep answers aligned with GPDB‚Äôs release cadence.
- **üåå Aurora Chat Surface** ‚Äî Animated, glassmorphism UI with live model telemetry for an immersive assistant experience.
- **üß© Model Context Protocol** ‚Äî Plug in new tools over HTTP SSE; the UI includes a control center for configuring MCP servers on the fly.
- **üì° Version Telemetry** ‚Äî Auto-detects connected Greenplum/Postgres versions and threads that information into every conversation.
- **üìä Production Friendly** ‚Äî Flyway migrations, Micrometer metrics, Prometheus endpoint, and resilient HTTP clients are all baked in.
- **üîê Zero-leak Secrets** ‚Äî API keys and encryption material are vaulted behind environment variables and AES-256-GCM (see `EncryptionService`).

## üé® Front-End Experience Deep Dive

The chat surface lives in `src/main/resources/static/` and is intentionally framework-free for speed and control.

### UI Stack at a Glance

| Layer | Significant Libraries / APIs | What They Bring |
|-------|------------------------------|------------------|
| `chat.css` | CSS custom properties, `@keyframes`, backdrop-filter, CSS grid/flexbox | Aurora gradients, glassmorphism panels, responsive layout, glowing status chips. |
| `chat.js` | Vanilla JS, `fetch`, `localStorage`, HTML template cloning, dynamic Markdown renderer | Streaming-style chat bubbles, conversation persistence, JSON-to-table rendering, live config badges. |
| `settings.js` | Fetch API, async/await, DOM diffing, optimistic UI updates | MCP server dashboard (add/edit/delete/test), API key visibility toggle, animated modals. |
| HTML (`index.html`) | Semantic layout, SVG/emoji iconography, template slots | Accessible structure, quick templating for message cards and sidebars. |

### Notable UX Details

- **Liquid Glass Palette** ‚Äî CSS variables define neon cyan, violet, and teal accents; `backdrop-filter` delivers frosted glass panels.
- **Markdown-with-tables Renderer** ‚Äî `renderMarkdown` in `chat.js` parses fenced code, inline code, and auto-transforms JSON arrays into data tables.
- **Conversation Memory** ‚Äî Local storage retains a durable `conversationId`, keeping Spring AI‚Äôs memory threads intact across reloads.
- **Telemetry Pill** ‚Äî Real-time label shows whether you are in local mode or OpenAI mode along with the resolved base URLs.
- **MCP Control Center** ‚Äî Settings modal lists tool servers with health indicators, last-tested timestamps, activation toggles, and encrypted-key handling.
- **Accessibility Hooks** ‚Äî Focus rings, keyboard shortcuts (`Enter` vs `Shift+Enter`), and reduced motion fallbacks keep the UI usable beyond the visuals.

## üß† Back-End & AI Brain

- **Spring Boot 3.3.6 + Java 21** ‚Äî Modern baseline with virtual-thread friendly configuration and native record support.
- **Spring AI 1.1.0-SNAPSHOT** ‚Äî ChatClient + Advisor pipeline with prompt templates in `src/main/resources/prompts/`.
- **pgvector** ‚Äî IVFFlat indexing (`V2__create_mcp_servers_table.sql` and `gp_docs` schema) enables fast semantic search.
- **MCP Integration** ‚Äî `DynamicMcpClientManager` + `AuthorizingHttpClient` manage per-server encryption and HTTP SSE tool streams.
- **Docs Chat Orchestration** ‚Äî `DocsChatService` wraps retrieval, prompt assembly, and answer formatting.
- **Resilience** ‚Äî `ResilientMcpToolCallbackConfig` uses resilience4j-like backoff to keep tool calls stable under failure.

## üöÄ Run It Now

> Requires Java 21+, a database with `pgvector`, and optionally an OpenAI-compatible endpoint.

```bash
# 1. (Optional) Capture secrets in .env
cp .env.template .env

# 2. Fire it up
./run.sh -c         # clean build + run (or run.bat on Windows)

# Subsequent runs
./run.sh

# Want OpenAI? Export a key first
export OPENAI_API_KEY=sk-your-key
./run.sh
```

Visit `http://localhost:8080` for the chat UI or `http://localhost:8080/admin/ingest` to load docs manually.

## üîÅ RAG & Data Flow

1. **Ingest** ‚Äî `DocsIngestor` pulls the official PDF (`app.docs.pdf-url`), chunks via `TokenTextSplitter`, generates embeddings, and stores them in `gp_docs`.
2. **Retrieve** ‚Äî `DocsChatService` performs vector similarity with `top-k` + `similarity-threshold` gates.
3. **Prompt** ‚Äî `PromptLoader` injects context (connected version, baselines, schema hints) into `gp_system.txt`.
4. **Respond** ‚Äî Spring AI ChatClient coordinates with either your local gateway or OpenAI. Responses feed the UI markdown renderer.
5. **Tool Call (optional)** ‚Äî When MCP is enabled, the assistant can call configured tools for live database metadata before answering.

Performance tips:
- Adjust ANN `lists` in `V2__create_mcp_servers_table.sql` when scaling past 100k chunks.
- Consider disabling the index during bulk ingestion, then re-creating.
- Run `ANALYZE gp_docs` after large ingest batches.

## üîê Security Checklist

- üîë Provide `APP_SECURITY_ENCRYPTION_KEY` before enabling the MCP settings UI ‚Äî encrypted API keys never touch disk in plain text.
- üß± Deploy behind TLS (reverse proxy or Spring Boot native SSL) ‚Äî sample config assumes HTTPS in production.
- üßæ Externalize secrets (`.env`, Vault, Kubernetes secrets) and never commit `.env` files.
- üõ°Ô∏è Add authentication (Spring Security / OAuth) before exposing endpoints outside a trusted network.
- üïµÔ∏è Enable database least privilege: ingestion user needs `CREATE EXTENSION` and CRUD on `gp_docs`, nothing more.
